## MT5 ファイル配置
1. MT5の「ファイル→データフォルダを開く」を実行
2. `MQL5/Experts/Mochipoyo/MochipoyoEA.mq5` を配置
3. `MQL5/Include/Mochipoyo/` に 3つの .mqh を配置
4. MetaEditor で `MochipoyoEA.mq5` を開き F7 でコンパイル（エラー0・警告0）

## コンパイル確認
- エラー0・警告0 が合格ライン
- コンパイル後は MT5 のエキスパートタブでログを確認する

### H4判定の現行仕様（v1.2・現行基準）
**現行コードは v1.2**。H4判定はダウ理論スイング方式（lookback=50）：
- 上昇 = `IsHigherHighHigherLow(symbol, PERIOD_H4, 50)`（高値・安値切り上げ）
- 下降 = `IsLowerHighLowerLow(symbol, PERIOD_H4, 50)`（高値・安値切り下げ）
- **実装ファイル**: `Include/MochipoyoSignal.mqh` 行266-279 付近

### v1.3（ハイブリッド判定）は❌不採用
v1.3は「窓化スイング AND EMA」ハイブリッド方式を提案したが、**検証の結果 不採用**。コードはv1.2に戻済み。
- GBPJPY下落相場(009b)では有効: PF 0.83→1.07 / ロング勝率 34.92%→45.57%
- **しかしGOLD全相場5年(009c)では収益改善ゼロ・リスク悪化**（v1.2との真の同条件比較）:
  - PF 1.09→1.08 / 純益 +$2,893→+$2,781 / 最大DD 19.46%→26.01% / 最長保持 157h→357h
- 結論: 改善はペア/相場依存。汎用採用は見送り、GBPJPY下落限定の選択肢として保留
- 詳細: `backtests/v1.2-baseline-5year/notes.md` / 仕様書 §8

## 既知のバグ・改善課題

### BUG-003: ポジション保持時間の長時間化（優先度：中）
**状況（v1.2/v1.3とも5年連続・GOLD・実測で確定）**:
- v1.2実測値: 157:44:40（≈6.5日） / 最大DD 19.46%
- v1.3実測値: 357:59:40（≈15日） / 最大DD 26.01%
- **帰因判定済み**: 同一の5年連続手法でもv1.2は157h止まり → 357hは測定アーティファクトでなく**v1.3ロジック由来の劣化**。v1.3不採用によりDD/保持の悪化分は解消。

**現状（v1.2基準）**: 最長157h・DD19.46%は許容範囲だが、48h超保持はv1.2にも残存。
**原因推定**: 実装レベルで保持時間の上限ロジックが存在しない。
**対応予定（任意・優先度中）**: v1.2に48時間保持上限ロジックを導入しDD改善を検証（次フェーズ候補）。
