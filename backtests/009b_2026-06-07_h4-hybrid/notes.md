## 修正内容

### 根本原因（BUG-004）
- 従来の FindSwingHighs/FindSwingLows は「直近の2スイング点」のみを抽出
- 下落相場の小反発でも局所的な「高値切り上げ・安値切り上げ」を作り、IsHigherHighHigherLow() が TRUE を返す
- 結果：下落相場でロング取引が過剰に許可される（ロング勝率33.3%）

### v1.3 解決策：ハイブリッド判定
```
H4上昇 = [窓化スイング上昇] AND [EMA上昇]
H4下降 = [窓化スイング下降] AND [EMA下降]
```

#### 窓化スイング（原因①③を直接解消）
- lookback 50本を前半(3-25本)・後半(25-50本)に分割
- 各窓の極値(最高値・最安値)を抽出して比較
- 直近窓が過去窓を超えて初めて「切り上げ」と判定
- 小反発では過去窓を超えられない → 偽トレンド検出が排除される

#### EMA条件（押し目反発の偽ロングを確実に弾く）
- EMA20 > EMA40 かつ EMA40が上向き(3本前比) → 上昇
- その鏡像 → 下降
- 下落相場では EMA40が下向きなので、ロングは自動的にブロック

### 分析・考察

**予想と異なる形で成功。** v1.3は「下落相場でロングをブロックして件数を減らす」想定だったが、
実際は**ロング件数が増加(63→79)しつつ勝率が大幅改善(34.92%→45.57%)**した。

- **v1.3の効果**: 新H4判定（窓化スイング＋EMA）は、ロングを減らすのではなく**質の高いロングを選別**した。
  下落相場の中の「本物の上昇押し目」を捉え、ダマシを除外。結果、負け年(-$988)を勝ち年(+$408)に転換、
  DDも半減(16.33%→7.95%)、シャープもプラス転換。下落相場での機能不全(BUG-004)は実質的に解消。
- **予期しない変化①**: ロング件数の増加。「件数≤50」の目標は設計仮説に基づくもので、本質的な目的
  （下落相場で利益を出す）は達成。件数目標は撤回し、勝率・PF・純益で評価すべき。
- **予期しない変化②**: 最長保持時間が139h→357h(約15日)に悪化。BUG-003が顕在化。要対応。
- **次のアクション**:
  1. **回帰テスト必須**（010-012: 上昇004 / レンジ005 / 上下動006 など）。
     下落相場で改善した分、他相場で悪化していないかを必ず確認してから v1.3 を確定する。
  2. BUG-003（保持時間上限48h）の検討。
  3. 全期間で改善が確認できたら、合言葉「アルゴリズム保存」でv1.3をスナップショット化。
