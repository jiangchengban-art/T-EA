# MochipoyoEA セットアップガイド

## 1. MT5 データフォルダを開く

MetaTrader5 を起動し、メニューから「ファイル」→「データフォルダを開く」をクリック。

## 2. ファイルを配置する

### なぜファイルを分ける必要があるのか？

MetaTrader5 では、EA を動かすために **EA ファイル** と **ロジックファイル** を別々のフォルダに配置する必要があります。これは MT5 のルールです。

- **EA ファイル** (`MochipoyoEA.mq5`)：チャートにアタッチして実行される本体
- **ロジックファイル** (`.mqh` ファイル)：リスク計算、スコアリング、インジケーター計算などの機能パーツ

2つを別フォルダに分けることで、EA が自動的に必要なロジックを「引き込んで」実行できます。

### 配置手順

**ステップ 1: 必要なフォルダを作成する**

データフォルダを開いた後、以下の 2 つのフォルダを作成してください：

```
MQL5/
├── Experts/
│   └── Mochipoyo/           ← ここを作成（EA 本体を置く場所）
└── Include/
    └── Mochipoyo/           ← ここを作成（ロジックファイルを置く場所）
```

**ステップ 2: ファイルをコピーする**

このリポジトリから MT5 データフォルダへ以下のようにコピー：

| このリポジトリ | → | MT5 の場所 | 目的 |
|---|---|---|---|
| `MochipoyoEA.mq5` | → | `MQL5\Experts\Mochipoyo\MochipoyoEA.mq5` | EA 本体（チャートに乗せるやつ） |
| `Include\MochipoyoMTF.mqh` | → | `MQL5\Include\Mochipoyo\MochipoyoMTF.mqh` | インジケーター値を取得するロジック |
| `Include\MochipoyoSignal.mqh` | → | `MQL5\Include\Mochipoyo\MochipoyoSignal.mqh` | スコアリング・判定ロジック |
| `Include\MochipoyoRisk.mqh` | → | `MQL5\Include\Mochipoyo\MochipoyoRisk.mqh` | ロット計算・リスク管理ロジック |

**よくある間違い：**
- ❌ すべてを同じフォルダに入れる → コンパイルエラーになります
- ❌ `Mochipoyo` フォルダを作らず、`MQL5\Include\` に直接入れる → EA が見つけられません
- ✅ **正しい：** `Experts` と `Include` の両方に `Mochipoyo` サブフォルダを作って分ける

## 3. コンパイル（EA のエラーチェック）

配置したファイルが正しく読み込まれるか確認します。

### 手順

1. **MetaEditor を起動**
   - MT5 で F4 キーを押すか、メニュー「ツール」→「MetaEditor」

2. **MochipoyoEA.mq5 を開く**
   - MetaEditor の「ファイル」→「開く」から `MQL5\Experts\Mochipoyo\MochipoyoEA.mq5` を選択
   - または、左パネルの「ファイル」ツリーから探してダブルクリック

3. **F7 キーでコンパイル**
   - F7 を押す、または上部ツールバーの「コンパイル」ボタンをクリック

4. **ログを確認**
   - 画面下部のログ欄を見て、以下のどちらかを確認：
     - ✅ **成功：** 「0 errors, 0 warnings」と表示 → そのまま次へ進む
     - ❌ **失敗：** エラーメッセージが表示 → ファイル配置を再確認してください

### もしエラーが出た場合

「Cannot open file」などのエラーが出た場合、ファイル配置をチェック：
- `MQL5\Include\Mochipoyo\` フォルダに 3 つの `.mqh` ファイルがすべて入っているか
- ファイル名に日本語やスペースが混ざっていないか
- パスが Windows のバックスラッシュ `\` になっているか（スラッシュ `/` ではなく）

## 4. EA をチャートにアタッチ（実際に動かす準備）

コンパイル成功後、EA をチャートに乗せて実行します。

### 手順

1. **MT5 のナビゲーターペインを開く**
   - 左側に「ナビゲーター」パネルがあります（なければ Ctrl+N）
   - 「Expert Advisors」の三角をクリックして展開

2. **MochipoyoEA を探してダブルクリック**
   - ナビゲーターに「Mochipoyo」→「MochipoyoEA」が見えます
   - ダブルクリックするか、対象チャートにドラッグ&ドロップ

3. **対象チャートを開く（重要）**
   - **必ず M5 チャート** で実行してください（他の時間足では動きません）
   - **対象通貨ペア：** GBPJPY / GBPUSD / XAUUSD のいずれか
   - チャートを開いてから EA をアタッチしてください

4. **EA 設定ダイアログで許可する**
   - EA が起動すると、「エキスパートアドバイザ」ダイアログが表示
   - 「**自動売買**」にチェック ✅
   - 「**DLL のインポートを許可**」にチェック ✅（必要に応じて）
   - 「OK」をクリック

5. **自動売買ボタンを確認**
   - MT5 の右上ツールバーに「自動売買」ボタンがあります
   - ボタンが **緑色 🟢** になっていれば、EA が動作中です
   - 灰色 ⚪ なら動いていません

### ここが大事！

- **デモ口座で最初は必ずテスト** してください
- 実口座に投入する前に、最低 3 ヶ月間のフォワードテストを実施
- リスク設定は 1% を守ること

## 5. 主要パラメーター（チューニング用）

EA をアタッチするときに、パラメーター値を調整できます。初心者は **既定値のままで OK** です。

| パラメーター | 既定値 | 説明 | 注意 |
|---|---|---|---|
| **InpRiskPercent** | 1.0 | 1 トレードで口座残高の何 % をリスクにするか | **1 を超えたら危険** |
| **InpMinScore** | 8 | エントリーに必要な最小スコア（12 点満点） | 8 以上で発注。低いと多く取引 |
| **InpTargetRR** | 1.2 | 利益確定時の目標リスク・リワード比 | 1.2 = SL 幅の 1.2 倍を TP とする |
| InpMinRR | 1.0 | エントリー前の最小 RR 許容値 | これより低い RR なら発注しない |
| InpTrailByEMA40 | false | EMA40 に追従するトレーリング（上級者向け） | true でポジションの SL を自動追従 |
| InpBufferPips | 5 | 損切りラインの安全マージン（pips） | GOLD は 0.5$ 固定 |
| InpMaxSpreadPoints | 50 | スプレッドが大きすぎる場合は発注しない（ポイント） | 50 を超えるスプレッドなら待機 |
| InpNewsAvoidHours | （空） | 指定時間帯のトレードをスキップ | 例：`Mon21:30-22:30;Wed15:00-15:30` |

### 初心者向けの設定案

- **保守的：** `InpMinScore = 9` にして、より厳しい条件だけでエントリー
- **攻撃的：** `InpMinScore = 7` にして、シグナルを多く取る（リスク増加）
- **リスク調整：** `InpRiskPercent = 0.5` で、1 トレード 0.5% リスクに下げる

## ⚠️ 注意事項（これを守らないと資金が消えます）

### 実口座を使う前に

- **デモ口座で最低 3 ヶ月間のフォワードテストを必ず実施**
  - 過去テスト（バックテスト）は机上の空論です
  - 実際の市場変動で EA がどう動くか確認してください

### リスク管理

- **1 トレードのリスクは口座残高の 1% 以下**（`InpRiskPercent = 1.0` 厳守）
  - 例：口座が 1 万円 → 1 トレードで最大 100 円のリスク
  - 1% を超えたら、すぐに EA を止めてください

- **ナンピン禁止、両建て禁止**
  - 1 つのシンボル（GBPJPY など）では、同時に 1 ポジションだけ持つ
  - 負けポジションを追加で買い増ししない

### トレード前のチェックリスト

- ✅ ファイルが正しい場所に配置されている
- ✅ コンパイルで「0 errors, 0 warnings」が出た
- ✅ チャート時間足が M5 になっている
- ✅ 通貨ペアが GBPJPY / GBPUSD / XAUUSD のいずれかである
- ✅ EA の「自動売買」が緑色 🟢 になっている
- ✅ デモ口座で十分なテストを終わっている
